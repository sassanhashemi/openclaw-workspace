## Usage Instructions

### For OpenClaw

1. **On Project Start:** Copy this template to new project directory as `CLAUDE.md`
2. **Customize:** Fill in all bracketed [placeholders] with project specifics
3. **Keep Updated:** Update metrics, insights, decisions as project progresses
4. **Auto-Update:** Update "Last Updated" timestamp whenever file is modified

### For You (Sassan)

1. **Review weekly:** Check that OpenClaw is keeping this file current
2. **Add decisions:** Major strategic decisions should be logged in Decision Log
3. **Update criteria:** Adjust success criteria if validation reveals need
4. **Archive when complete:** Move to archive folder with final status

### What OpenClaw Will Update Automatically

- Metrics Dashboard (daily/weekly)
- Insights & Patterns (as discovered)
- Key Contacts table (after each interview)
- Experiments (status and results)
- Last Updated timestamp (on every change)

### What You Update Manually

- Project Context (if strategy changes)
- Decision Log (for strategic decisions)
- Phase transitions (go/no-go decisions)
- Autonomy adjustments (if policies change)
